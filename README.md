# 🥗 Calorie Bot — Чат-бот для подсчёта КБЖУ в ВКонтакте

Простой и удобный бот для отслеживания питания прямо в диалоге ВКонтакте. Считает калории, белки, жиры и углеводы, помогает достигать целей по весу.

---

## 📌 Возможности

###  Для пользователя
- ✅ **Пошаговый онбординг** — расчёт суточной нормы КБЖУ по формуле Миффлина-Сан Жеора
- ✅ **Добавление еды** — по названию, весу или из списка
- ✅ **Нечёткий поиск** — понимает опечатки («греча» → «Гречка»)
- ✅ **Личные продукты** — добавление своих продуктов с упаковки
- ✅ **Статистика за день** — с прогрессом к цели и детализацией
- ✅ **Удаление записей** — исправление ошибок в дневнике
- ✅ **FAQ** — встроенная справка

### 🔐 Для администратора
- 📊 **Общая статистика** — сводка по всем пользователям за день
- 📋 **Детальные отчёты** — по каждому пользователю
- 👥 **Список пользователей** — все зарегистрированные в боте
- 🔔 **Уведомления в реальном времени** — при каждом добавлении/удалении еды

##  Установка и запуск

### 1. Клонирование и подготовка

    cd C:\projects
    mkdir calorieBot
    cd calorieBot

    python -m venv venv
    venv\Scripts\activate          # Windows
    # source venv/bin/activate     # macOS/Linux

### 2. Установка зависимостей

    pip install vk-api sqlalchemy thefuzz python-Levenshtein python-dotenv
    pip freeze > requirements.txt

### 3. Настройка ВКонтакте

1. Создайте сообщество ВКонтакте (или используйте существующее)
2. **Управление → Сообщения** → включите **«Сообщения сообществ»**
3. **Настройки для бота** → включите **«Возможности ботов»**
4. **Управление → Работа с API → Long Poll API** → включите, выберите версию **5.199**
5. Включите событие **«Входящее сообщение»**
6. **Ключи доступа** → создайте ключ с правами:
   - ✅ Сообщения сообщества
   - ✅ Управление сообществом
   - ✅ Фотографии

### 4. Создание файла `.env`

    VK_GROUP_TOKEN=vk1.a.XXXXXXXXXXXXXXXXXXXXXXXX
    VK_GROUP_ID=123456789
    ADMIN_ID=987654321

> `VK_GROUP_ID` — ID сообщества (число из URL `vk.com/club123456789`)
> `ADMIN_ID` — ваш личный VK ID (для получения отчётов)

### 5. Инициализация базы данных

    python init_db.py

Ожидаемый вывод:

    🔄 Создание таблиц в базе данных...
    ✅ Таблицы созданы
    📂 Загрузка продуктов из data/products.csv...
    ✅ Загружено 12 продуктов в базу данных

### 6. Запуск бота

    python main.py

Ожидаемый вывод:

    Бот запущен. Группа ID: 123456789
    Ожидание сообщений...

## 🗄 База данных

Проект использует **SQLite** + **SQLAlchemy**. Файл `kbju_bot.db` создаётся автоматически.

### Таблицы

| Таблица | Описание |
|---------|----------|
| `users` | Профили пользователей (цели, норма КБЖУ) |
| `global_products` | Общая база продуктов (12 шт.) |
| `user_products` | Личные продукты пользователей |
| `daily_logs` | Дневник питания (записи за день) |

### Добавление продуктов в базу

Отредактируйте `data/products.csv`:

    name,calories,proteins,fats,carbs
    Гречка ядрица (сухая),313,12.6,3.3,62.1
    Рис белый (сухой),344,6.7,0.7,78.9

Затем пересоздайте БД:

    del kbju_bot.db
    python init_db.py

##  Как считается норма

Используется **формула Миффлина-Сан Жеора**:

    Мужчины: BMR = 10 × вес + 6.25 × рост − 5 × возраст + 5
    Женщины: BMR = 10 × вес + 6.25 × рост − 5 × возраст − 161

    TDEE = BMR × коэффициент активности

### Коэффициенты активности

| Уровень | Коэффициент |
|---------|-------------|
| Минимальная (сидячий) | 1.2 |
| Слабая (1-3 раза/нед) | 1.375 |
| Умеренная (3-5 раз/нед) | 1.55 |
| Высокая (6-7 раз/нед) | 1.725 |
| Экстремальная (2 раза/день) | 1.9 |

### Корректировка под цель

- **Похудение**: −20% от нормы
- **Поддержание**: без изменений
- **Набор массы**: +15% от нормы

### Распределение БЖУ

- Белки: 2 г на кг веса
- Жиры: 1 г на кг веса
- Углеводы: остаток калорий
